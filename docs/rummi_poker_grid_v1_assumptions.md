# Rummi Poker Grid — v1 구현 가정

> 기획 확정 반영. 변경 시 `rummi_poker_grid_game_logic.md`·코드·테스트를 함께 수정한다.  
> **진행·승리 프레임**: **목표 \(T\)·버림 \(D\)** — 확정 횟수 무제한. **죽은 줄은 보드 버림으로만 완화**. **만료**: 25칸 만재 또는 **현재 덱 소진**(`copiesPerTile` 기반, GDD §8.4).

| 항목 | 확정 내용 |
|------|-----------|
| 평가 라인 | **행 5 + 열 5 + 주·반 대각 2 = 12줄** (빙고식) |
| 빈 칸이 있는 줄 | **판정 안 함** — 5칸이 모두 찰 때만 `HandEvaluator` 적용 |
| Straight | 일반 연속 `n..n+4`(1≤n≤9) + 휠 **10–11–12–13–1** (A=1) |
| 죽은 줄 | **하이카드·원페어** — 양의 점수 없음 (`isDeadLineRank`). **방치·대기 ±** 는 미확정 — `LineHazardTuning` 기본 **0** |
| 일괄 확정 | **완성 줄 개수 제한 없음**. **전용 버튼 또는 임의 완성줄 탭** → 대기 완성 줄 **전부** 한 번에 점수·제거 |
| 조커 | **메타(장착 슬롯)만** — 보드 타일 모델에 포함하지 않음 (GDD §7) |
| 줄 제거 | 5칸 완성 줄은 **수동 일괄 확정** (GDD §4.3). 족보와 별개로 `gddCanClearLine`은 true |
| 덱·만료 | `copiesPerTile` 기반(`4×13×copiesPerTile`); 구현은 **52/104 공용**. **전부 소모** 시 만료(GDD §2.2·§8.4). 리셔플 없음 |
