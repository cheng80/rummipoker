# Rummi Poker Grid — v2 전투/점수 시스템 제안서

## 문서 목적
이 문서는 기존 **5칸 완성 강제 + 완성 줄만 평가** 구조를 폐기하고,
**즉시 확정(Instant Confirm) + 겹침(Overlap) 중심 점수 시스템**으로 전환하기 위한 구현 기준 문서다.

이 문서는 실제 개발 진행용 기준 문서이며, 이후 구현은 본 문서를 우선 기준으로 삼는다.
충돌 시에는 본 문서를 먼저 갱신한 뒤 코드와 하위 문서를 맞춘다.

---

## 1. 설계 목표

### 1.1 목표
기존 구조에서 발생하던 아래 문제를 완화한다.

- 투페어·트리플 등 일부 족보를 만들기 위해 **의미 없는 추가 카드 1장**을 강제로 채워야 함
- 줄 완성 강제로 인해 보드에 **잔여 카드가 누적**되고, 뒤로 갈수록 확정이 더 어려워짐
- 블라인드 승리가 목적임에도, 기본 스테이지 템포가 느리고 덱 소모가 큼
- 겹치는 줄을 활용하는 전략은 좋지만, 현재는 **5칸 강제 완성**이 전략보다 제약으로 더 크게 작용함

### 1.2 새 방향
새 시스템은 아래 방향으로 전환한다.

- **5칸 완성 강제 제거**
- **확정 버튼을 누른 시점의 성립 족보를 즉시 점수화**
- **겹치는 라인에 동시에 기여한 타일에 보너스 부여**
- 초반에는 기본 족보로 점수를 확보하고,
- 후반에는 겹침 + 제스터 + 조건형 증폭으로 블라인드를 넘기는 구조를 만든다.

## 2. 핵심 규칙 변경

### 2.1 제거되는 규칙
아래 규칙은 제거한다.

- **5칸이 모두 찬 줄만 평가**
- **줄 완성 대기 상태**
- **완성 줄만 확정 가능**

### 2.2 새 규칙
아래 규칙을 새 기준으로 사용한다.

- 플레이어는 **언제든 확정 버튼**을 누를 수 있다.
- 확정 버튼을 누르면 현재 보드에서 **점수 성립 가능한 모든 라인**을 즉시 평가한다.
- 각 라인은 현재 놓인 카드들만으로 판정한다.
- 5칸이 모두 차지 않았더라도, **현재 놓인 카드로 유효 족보가 성립하면 점수화 가능**하다.
- 점수화된 라인에서 **족보 성립에 실제 기여한 카드만 제거**한다.
- 제거되지 않은 카드는 보드에 남아 다음 확정의 기반이 된다.

## 3. 평가 라인 범위
평가 라인 자체는 기존과 동일하게 유지한다.

- 가로 5줄
- 세로 5줄
- 주 대각선 1줄
- 반 대각선 1줄

즉, 총 **12줄**을 평가 대상으로 사용한다.

---

## 4. 라인 평가 방식

### 4.1 기본 원칙
각 라인은 현재 그 줄 위에 존재하는 카드들만으로 평가한다.

예를 들어 한 줄에 카드가 3장만 있어도:

- 그 3장으로 트리플이 성립하면 점수화 가능
- 그 3장으로 아무 유효 족보가 없으면 점수 없음

### 4.2 라인별 최고 족보 1개만 선택
각 라인은 **현재 카드 상태에서 성립 가능한 최고 족보 1개만** 선택한다.
한 라인에서 복수 족보를 동시에 따로 점수화하지 않는다.

예:
- 같은 4장으로 트리플과 원페어 해석이 모두 가능해도, 더 높은 족보 1개만 채택
- 5장 상태에서 스트레이트와 플러시가 동시에 되면 스트레이트 플러시 1개만 사용

### 4.3 카드 수별 성립 가능 족보
카드 수에 따라 허용 가능한 족보는 아래처럼 본다.

| 카드 수 | 성립 가능 족보 |
|---|---|
| 1 | 없음 |
| 2 | One Pair |
| 3 | Three of a Kind |
| 4 | Two Pair, Four of a Kind |
| 5 | 기존 전체 족보 |

### 4.4 스트레이트 / 플러시의 의미
- **스트레이트 / 플러시 / 풀하우스 / 스트레이트플러시**는 여전히 **5장일 때만** 성립한다.
- 따라서 5장의 의미는 완전히 사라지지 않는다.
- 다만 모든 족보가 5장 강제를 받지 않도록 바꾼다.

즉:
- 투페어, 트리플, 포카드는 적은 장수로도 점수화 가능
- 스트레이트, 플러시 계열은 여전히 5장의 의미를 가진다

## 5. 족보 및 기본 점수
기존 점수표를 유지하되, 현재 최종 룰에서는 **원페어는 판정만 하고 점수는 부여하지 않는다**.
즉, 하이카드와 원페어는 모두 점수 없음이다.

| 족보 | 기본 점수 | 비고 |
|---|---:|---|
| High Card | 0 | 점수 없음 |
| One Pair | 0 | 판정은 하지만 dead line |
| Two Pair | 25 | 유지 |
| Three of a Kind | 40 | 유지 |
| Straight | 70 | 5장만 성립 |
| Flush | 50 | 5장만 성립 |
| Full House | 80 | 5장만 성립 |
| Four of a Kind | 100 | 유지 |
| Straight Flush | 150 | 5장만 성립 |

### 5.1 원페어 현재 처리
원페어는 **부분 줄 평가 대상**에는 포함되지만, 현재 구현에서는 **dead line**으로 본다.

즉:

- 2장/3장/4장 줄에서도 원페어 판정 자체는 유지
- 하지만 점수는 `0`
- 확정 후보에는 포함되지 않음
- 화면상 점수 줄 체크도 하지 않음

이 설계는 나중에 밸런스 실험으로 다시 점수화할 여지를 남기기 위해, **판정 enum은 유지하고 점수/확정만 막는 구조**를 택한다.

## 6. 겹침 보너스 시스템

### 6.1 정의
하나의 타일이 확정 시점에서 **여러 라인의 점수 성립에 동시에 기여**할 수 있다.
이때 해당 타일의 겹침 기여도를 계산해 라인 점수에 보너스를 준다.

### 6.2 k의 정의
- `k = 해당 타일이 이번 확정에서 점수 성립에 기여한 라인 수`

예:
- 어떤 타일이 한 줄의 트리플에만 기여 → `k = 1`
- 어떤 타일이 가로 투페어와 세로 투페어 둘 다의 성립 카드로 사용 → `k = 2`
- 가로/세로/대각 3개 라인에 동시에 기여 → `k = 3`

### 6.3 보너스 공식
겹침 보너스는 아래 공식을 사용한다.

```text
multiplier = min(1 + α × (k - 1), cap)
```

기본값:

- `α = 0.3`
- `cap = 2.0`

### 6.4 k별 multiplier 예시
| k | multiplier |
|---|---:|
| 1 | 1.0 |
| 2 | 1.3 |
| 3 | 1.6 |
| 4 | 1.9 |
| 5 이상 | 2.0 |

### 6.5 적용 방식
라인 점수 계산 시, 그 라인에 기여한 카드들의 `k`를 참고해 보너스를 적용한다.

구현 권장안:
- 한 라인의 성립 카드들 중 **가장 높은 k 값을 대표값으로 사용**한다.

즉:

```text
lineScore = baseScore × overlapMultiplier
```

여기서 `overlapMultiplier`는 해당 라인 성립 카드들 중 최고 `k` 기준으로 계산한다.

### 6.6 구현 단순화 권장
초기 구현은 복잡한 평균값/합산 방식보다 아래 방식으로 고정한다.

- 각 라인의 성립 카드 집합 계산
- 각 카드가 이번 확정에서 몇 개 라인에 기여했는지 계산
- 그 라인의 성립 카드 중 최고 `k`를 찾아 multiplier 계산
- 그 값을 라인 전체 점수에 곱함

이 방식은 구현이 단순하고, 겹침의 존재감을 분명히 보여 준다.

## 7. 확정 처리 흐름
플레이어가 확정 버튼을 누르면 아래 순서로 처리한다.

### 7.1 처리 순서
1. 현재 보드 기준으로 12개 라인 스캔
2. 각 라인에서 현재 카드들로 가능한 최고 족보 1개 선택
3. 점수가 1 이상인 라인만 확정 후보에 포함
4. 각 확정 후보 라인의 성립 카드 집합 계산
5. 전체 후보 라인을 기준으로 각 카드의 `k` 계산
6. 라인별 겹침 multiplier 적용
7. 라인 점수 합산
8. 제스터 보정 적용
9. 총 점수를 블라인드 누적 점수에 반영
10. 점수화된 라인들의 성립 카드 합집합을 제거
11. 빈 칸은 이후 드로우 규칙에 따라 보충

### 7.2 점수 없는 라인
- 하이카드와 원페어 라인은 확정 대상이 아님
- 점수 0 라인은 제거도 하지 않음

## 8. 제거 규칙
기본 철학은 기존과 동일하게 유지한다.

- **족보 성립에 기여한 카드만 제거**
- 성립에 직접 쓰이지 않은 카드는 남는다

### 8.1 족보별 제거 규칙
| 족보 | 제거 방식 |
|---|---|
| One Pair | 제거 없음 (현재 dead line) |
| Two Pair | 페어 4장 제거 |
| Three of a Kind | 트리플 3장 제거 |
| Four of a Kind | 포카드 4장 제거 |
| Straight | 5장 전부 제거 |
| Flush | 5장 전부 제거 |
| Full House | 5장 전부 제거 |
| Straight Flush | 5장 전부 제거 |

### 8.2 남는 카드의 의미
남는 카드는 다음 확정의 기반으로 남는다.
즉, 시스템 목표는 다음 둘을 동시에 만족시키는 것이다.

- 의미 없는 5번째 카드 강제는 없앤다
- 그러나 겹침/잔여 카드 기반 전략은 유지한다

---

## 9. 제스터 시스템 방향
기존 `chips_bonus / mult_bonus / xmult_bonus` 구조는 유지한다.
다만 새 시스템에서 제스터는 아래 축으로 재정렬한다.

### 9.1 역할 축
1. **기본 족보 강화**
   - 특정 족보에 chips/mult 추가
2. **겹침 강화**
   - `α` 증가
   - `cap` 증가
   - 특정 조건에서 `k + 1` 취급
3. **자원/상태 연동 강화**
   - 남은 디스카드
   - 덱 잔량
   - 상태형 누적값

### 9.2 의도
- 초반: 기본 족보 + 소규모 제스터 보정
- 중반: 겹침 설계가 의미를 가지기 시작
- 후반: 겹침 + 상태형 제스터 + 조건형 증폭으로 폭발

즉 후반 승리의 핵심은 **기본 족보 자체가 아니라, 족보를 어떻게 증폭시키는가**가 되어야 한다.

## 10. 블라인드 진행 의도
블라인드 승리가 목적이므로,
점수 시스템은 아래 흐름을 만들도록 설계한다.

### 10.1 초반
- 원페어 / 투페어 / 트리플로도 소블라인드를 넘길 수 있어야 함
- 즉시 확정으로 템포 확보

### 10.2 중반
- 겹치는 줄 배치가 단순 족보보다 더 높은 효율을 가짐
- 확정 타이밍과 보드 유지 판단이 중요해짐

### 10.3 후반
- 기본 족보만으로는 부족함
- 겹침 보너스 + 제스터 증폭 + 상태 누적을 통해 타겟 점수를 넘기는 구조를 의도함

## 12. 구현 기준 요약

### 12.1 HandEvaluator 변경
- 5장 전용 평가기에서 벗어나
- **2~5장 평가 지원** 필요
- 부분 카드 수에서도 성립 가능한 족보 판정 지원

### 12.2 Engine 변경
- 12줄 전체 스캔 유지
- 각 줄에 대해 현재 존재 카드만 모아 평가
- 0점 라인은 확정 후보 제외

### 12.3 Overlap 계산 추가
- 확정 후보 라인의 성립 카드 집합 계산
- 카드별 `기여 라인 수(k)` 계산
- 라인별 최고 `k` 기반 multiplier 적용

### 12.4 제거 처리
- 확정된 모든 라인의 성립 카드 합집합 제거
- 중복 제거 허용

## 14. 한 줄 요약
이 변경안은 게임을

**“5칸 완성 퍼즐”** 에서
**“즉시 확정 + 겹침 설계 + 후반 증폭”** 구조로 바꾸기 위한 기준 문서다.

핵심은 다음 4가지다.

- 5칸 강제 제거
- 원페어는 평가하되 현재는 점수화하지 않음
- 확정 버튼 즉시 점수화
- 겹침 보너스를 후반 폭발의 핵심 축으로 사용
