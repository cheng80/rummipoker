# Rummi Poker Grid — 게임 로직 (구현 스펙)

> 본 문서는 `docs/rummi_poker_grid_gdd.md`를 **코드 작성 기준**으로 풀어 쓴 것이다.  
> GDD와 충돌 시 **GDD를 먼저 수정**한 뒤 본 문서를 맞춘다.

---

## 1. 엔티티

### 1.1 타일 (Tile)

- **숫자** `n`: 정수, 범위 **1 ~ 13**
- **색** `c`: 네 가지 중 하나 — 🔴 🔵 🟡 ⚫ (구현: `enum TileColor { red, blue, yellow, black }`)
- 동일 `(n, c)` 조합은 루미큐브 규칙상 덱에 2장까지 존재할 수 있음 → **인스턴스 ID**(또는 `deckIndex`)로 구분할지는 덱 설계에서 결정

### 1.2 보드 (Board)

- **5 × 5** 고정 그리드
- **칸 상태**: `Empty | Occupied(TileInstance)`
- **이동 없음**, **중력 없음** — 배치는 “빈 칸에 한 장 놓기” 등 GDD에 따른 액션으로만 변함 (구체 배치 규칙은 GDD 보완 시 본 절 갱신)

### 1.3 웨이스트 (임시 슬롯, Waste)

- **최대 3칸** — 큐 또는 고정 슬롯으로 모델링
- 드로우된 카드가 먼저 들어가거나, 플레이어가 보드에 올리기 전 임시 보관용 (GDD 상세에 맞춰 `WasteState` 필드 정의)

### 1.4 덱 (Deck)

- 남은 카드의 스택; **버림 후 즉시 1장 보충**은 덱에서 `pop` 하는 이벤트로 표현
- 초기 구성(몇 장씩 몇 세트)은 밸런스 설계 — 별표 `docs/rummi_poker_grid_balance.md` 후보

## 2. 턴 상태 머신 (개요)

한 턴의 논리 순서는 GDD §3과 동일하게 유지한다.

```
[턴 시작]
  → 드로우 (덱 → 웨이스트 또는 보드 규칙에 맞게)
  → 플레이어 액션: { 배치 | 버림(즉시 보충) }
  → 보드 갱신
  → 핸드 분석 (모든 평가 라인에 대해)
  → (UI) 제거 후보 표시 → 플레이어가 라인 선택 시 제거 처리
[턴 종료]
```

- **버림**: 턴당 1회, 스테이지 총 횟수 상한(기본 4) — 카운터는 `StageState`에 보관
- 자동 제거 없음 — 제거는 **항상 플레이어 입력**

## 3. 평가 라인 (Hand scope)

**설계 결정 (구현 기본값)** — GDD에 명시가 없어 아래를 기본으로 두고, 기획 변경 시 본 절만 수정한다.

- **평가 단위**: **행 5개 + 열 5개 = 총 10줄**
- 각 줄은 정확히 **5장의 타일**을 본다 (빈 칸이 있으면 그 줄은 “완성된 5장 핸드”로 보지 않거나, GDD 별도 규칙 적용 — **미정이면** 빈 칸 포함 줄은 `High Card` 이하만 평가 불가로 처리하는 방안을 택할 수 있음)

> ⚠️ GDD 확정 전까지: “빈 칸이 있는 줄의 핸드 타입”은 **반드시 기획 승인** 후 구현.

---

## 4. 포커 핸드 판정

한 줄에 놓인 5장에 대해 **가장 높은 족보 하나**를 선택하거나, GDD가 요구하는 방식(예: 최고 점수 조합만 인정)을 따른다. 기본은 **최상위 족보 1개**.

| 족보 | 조건 (5장 기준) | 점수 | 제거 시 점수 |
|------|-----------------|------|--------------|
| High Card | 위 족보 해당 없음 | 0 | 제거 불가(라인) |
| One Pair | 동일 숫자 2장 | 0 | 제거 가능 |
| Two Pair | 페어 2개 | 25 | ✔ |
| Three of a Kind | 동일 숫자 3장 | 40 | ✔ |
| Straight | 연속 숫자 | 50 | ✔ |
| Flush | 동일 색 5장 | 60 | ✔ |
| Full House | 3 + 2 | 80 | ✔ |
| Four of a Kind | 동일 숫자 4장 + 키커 1 | 100 | ✔ |
| Straight Flush | 연속 + 동일 색 | 150 | ✔ |

### 4.1 세부 규칙 (구현 시 고정할 항목)

- **Straight**: A를 하이/로우로 쓸지(예: 1-2-3-4-5, 10-11-12-13-1) — **미정이면** 1~13 연속만 인정
- **Straight vs Straight Flush**: 후자가 우선; 색이 섞인 연속은 Straight만
- **Flush**: 5장 전부 같은 색

위 세부는 `HandEvaluator` 모듈의 단위 테스트로 고정한다.

## 5. 제거 (Clear line)

- 플레이어가 **특정 라인(행/열)** 을 선택했을 때만 실행
- 결과:
  - **유효 점수 핸드**: 표의 점수 획득 + 해당 줄의 타일 제거(빈 칸)
  - **One Pair**: 점수 0 + 줄 제거 가능
  - **죽은 줄 (No Pair 이하로 정의되는 경우)**: **-10** + 줄 제거 (GDD §4.4)

---

## 6. 점수·패널티 (확장)

GDD §6~7의 내용은 **게임 전역 상태**에 반영한다.

| 시스템 | 저장 위치 예시 |
|--------|----------------|
| 제거 전 점수 미리보기 | UI/Flame HUD — 엔진이 `LineEvaluation` 반환 |
| 유지 보너스 (1턴 x1.2, 2턴 x1.5) | `ScoreMultiplierState` — “유지” 정의가 확정되어야 함 |
| 죽은 줄 방치 패널티 (-10 / -20) | `LineRiskState` — 턴 카운터 per line |
| 조커 (최대 5장착, 배율) | `JokerLoadout` + 판정 시 배율 적용 — **타일 모델 확장** (`isJoker` 또는 별 타입) 필요 |
